# Optimize System Prompt Task

## Purpose
기존 시스템 프롬프트를 분석하여 환각 방지와 정확도 향상을 위한 개선안을 제시합니다.

## Analysis Process
1. **현재 프롬프트 구조 분석**
   - 역할 정의의 명확성
   - 지시사항의 구체성
   - 제약 조건의 효과성

2. **문제점 식별**
   - 모호한 표현
   - 상충되는 지시
   - 누락된 제약 조건

3. **개선 방향 설정**
   - 환각 방지 강화
   - 출처 명시 의무화
   - 추측 금지 명확화

4. **최적화 적용**
   - 구조적 개선
   - 명확한 우선순위
   - 검증 가능한 지시

## Output Format
```markdown
### 시스템 프롬프트 최적화 분석

**현재 프롬프트 평가:**
- 강점: [현재 프롬프트의 좋은 점]
- 약점: [개선이 필요한 부분]
- 환각 위험도: [Low/Medium/High]

**주요 개선 사항:**
1. [개선 포인트 1]
   - 문제: [현재 문제점]
   - 개선안: [구체적 개선 방법]

2. [개선 포인트 2]
   - 문제: [현재 문제점]
   - 개선안: [구체적 개선 방법]

**최적화된 프롬프트:**
```
[개선된 프롬프트 전문]
```

**예상 효과:**
- 환각 감소율: [예상 %]
- 출처 명시율: [예상 %]
- 답변 정확도: [예상 개선도]
```

# Design Research Prompt Task

## Purpose
웹 리서치 데이터를 효과적으로 활용하는 프롬프트 구조를 설계합니다.

## Design Principles
1. **리서치 데이터 우선순위화**
   - 리서치 정보를 최우선 참조
   - 일반 지식 사용 금지
   - 출처 기반 답변 강제

2. **명확한 경계 설정**
   - 리서치 데이터 구역 명시
   - 시스템 지시와 분리
   - 데이터 신뢰도 표시

3. **출력 형식 지정**
   - 출처 번호 필수 포함
   - 인용 형식 표준화
   - 신뢰도 레벨 표시

## Template Structure
```markdown
### 리서치 기반 프롬프트 템플릿

**[시스템 역할 정의]**
당신은 보드게임 룰 전문가입니다.

**[핵심 제약 조건]**
⚠️ 중요: 아래 제공된 리서치 정보만을 사용하여 답변하세요.
- 추측이나 일반적 지식 사용 금지
- 모든 답변에 출처 번호 필수 명시
- 확실하지 않은 정보는 언급하지 마세요

**[리서치 데이터]**
================
📚 웹 리서치 결과:
{research_summary}

출처:
1. {source_1}
2. {source_2}
3. {source_3}
================

**[답변 형식]**
- 각 문장 끝에 출처 번호 표시 [1], [2]
- 리서치에 없는 정보는 "확인된 정보 없음" 명시
- 여러 출처가 상충할 경우 모두 언급

**[사용자 질문]**
게임: {game_title}
질문: {user_question}
```

## Variations
- **간단한 질문용**: 짧고 직접적인 템플릿
- **복잡한 질문용**: 상세한 분석 포함 템플릿
- **다중 출처용**: 출처 간 비교 분석 템플릿

# Effective Prompt Patterns

## 환각 방지 패턴

### 1. Explicit Constraints Pattern
```
"다음 제약 조건을 반드시 준수하세요:
1. 제공된 정보만 사용
2. 추측 금지
3. 불확실한 경우 '정보 없음' 명시"
```

### 2. Source Attribution Pattern
```
"모든 주장은 다음 형식으로 출처를 명시하세요:
[주장 내용] (출처: [번호])"
```

### 3. Negative Instructions Pattern
```
"절대 하지 말아야 할 것:
- 일반적인 지식으로 답변
- 출처 없는 정보 제공
- 가능성이나 추측 언급"
```

## 구조화 패턴

### 1. Context-Instruction Separation
```
=== 컨텍스트 ===
[데이터]

=== 지시사항 ===
[명령]

=== 제약조건 ===
[제한사항]
```

### 2. Priority Hierarchy
```
우선순위:
1급 (필수): [핵심 지시]
2급 (중요): [보조 지시]
3급 (선택): [추가 옵션]
```

## 검증 패턴

### 1. Self-Check Pattern
```
"답변 전 다음을 확인하세요:
□ 모든 정보가 제공된 자료에 있는가?
□ 출처가 명시되었는가?
□ 추측이 포함되지 않았는가?"
```

### 2. Confidence Level Pattern
```
"각 답변에 신뢰도를 표시하세요:
[확실] - 출처에 명확히 명시됨
[추론] - 출처 정보 기반 논리적 추론
[불확실] - 정보 부족"
```

# Common Hallucination Triggers

## 주의해야 할 프롬프트 요소

### 1. 모호한 지시
**문제가 되는 표현:**
- "적절히 답변하세요"
- "일반적으로 설명하세요"
- "상식적으로 판단하세요"

**개선된 표현:**
- "제공된 정보만으로 답변하세요"
- "출처 [X]의 내용을 인용하여 설명하세요"
- "리서치 데이터에 명시된 규칙만 언급하세요"

### 2. 열린 질문
**문제가 되는 형식:**
- "어떻게 생각하시나요?"
- "추천해주세요"
- "가능한 방법은?"

**개선된 형식:**
- "제공된 자료에서 찾을 수 있는 방법은?"
- "출처에 명시된 옵션은?"
- "리서치 결과에 따르면?"

### 3. 추론 유도
**위험한 패턴:**
- "아마도", "대략", "일반적으로"
- "~일 것입니다", "~로 보입니다"
- "경험상", "보통은"

**안전한 패턴:**
- "출처 [X]에 따르면"
- "리서치 결과는 다음과 같습니다"
- "제공된 정보에는 [내용] 없음"

## 환각 방지 체크리스트

1. **명시적 금지어 포함**
   - ❌ 추측, 가정, 일반화
   - ❌ 경험, 통념, 상식
   - ❌ 아마도, 보통, 대개

2. **필수 포함 요소**
   - ✅ 출처 번호
   - ✅ 직접 인용
   - ✅ 정보 부재 시 명시

3. **구조적 보호장치**
   - 리서치 데이터 명확한 구분
   - 우선순위 명시
   - 자가 검증 단계
